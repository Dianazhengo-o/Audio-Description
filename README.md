# Audio-Description

# 刺客聶隱娘：基於 GPT-4o 之電影口述影像自動生成研究

本專案旨在利用多模態大型語言模型（Multimodal LLM）GPT-4o，針對電影《刺客聶隱娘》之特定片段自動生成口述影像（Audio Description, AD）。透過系統化的提示工程（Prompt Engineering），對比不同語言文化背景下之口述影像準則對模型生成品質的影響。

---

## 特色

* **自動化影像處理**：利用 `FFmpeg` 與 `OpenCV` 對原始影片進行時段切割與影格擷取。為優化 Token 消耗與運算效率，將影格寬度等比例縮放至 512 像素，並以每 2 秒 1 幀之頻率進行採樣。
* **提示工程實驗**：設計兩套系統提示詞（System Prompts）以評估模型表現：
    * **華語 AD 核心規範**：強調客觀性邊界、精煉動詞（如「掠」、「刺」）及東方寫意美學，並嚴格限制時間副詞之使用。
    * **國際英文 42 項準則**：導入詳盡的國際標準，包含動作優先級、避免身分猜測及提前播報等規範。
* **節奏與風格對照**：對比「人類劇情提示」對模型捕捉影片「從靜到動」節奏轉折（伏擊至刺殺結束）之影響。

## 技術架構 

* **開發環境**：Google Colab
* **主要語言**：Python
* **核心套件**：
    * `openai`：調用 GPT-4o 模型
    * `cv2 (OpenCV)`：影格擷取與圖像預處理
    * `ffmpeg`：影片片段精確切割

## 執行流程 

1. **影片預處理**：將 `demo.mp4` 切割為指定之 40 秒測試片段。
2. **影格編碼**：將擷取之影格轉換為 Base64 編碼，以符合 OpenAI API 之多模態輸入格式。
3. **模型推理**：輸入影像序列與對應之系統準則，要求模型輸出結構化之 JSON 資料。
4. **輸出分析**：
    * `sequence_analysis`：40 秒影像之節奏與動態分析。
    * `visual_features`：美學氛圍與光影特徵描述。
    * `ad_script`：最終生成之 120 至 150 字口述影像稿。
